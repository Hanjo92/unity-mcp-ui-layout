# Backlog

This backlog tracks likely next improvements for the repository after `v0.3.0`.

이 backlog는 `v0.3.0` 이후 저장소에서 이어갈 만한 개선 항목을 정리합니다.

## High Priority / 높은 우선순위

### 1. UI Toolkit Deepening

- add dedicated UI Toolkit failure patterns
- add UI Toolkit-specific examples
- clarify layout ownership and text behavior in flex-heavy screens

### 2. Naming and Folder Examples

- add concrete before/after examples for shared versus screen-owned assets
- show variant folder organization with a real prefab family example

### 3. Shared Asset Verification Recipes

- add concrete MCP-oriented review flow for checking another known usage before editing a shared base
- add example prompts for sprite/material/TMP-style safety checks

## Medium Priority / 중간 우선순위

### 4. Localization-Focused Examples

- add examples for long labels, multi-line body text, and number growth
- add one example for a screen that must survive both English and longer translated text

### 5. Mobile Device Profiles

- add explicit example verification sets for tall phones, tablets, and wider mobile ratios
- document when portrait-only assumptions are acceptable

### 6. Release Hygiene

- keep platform adapters synchronized with each significant skill release
- consider adding a short release checklist document

## Low Priority / 낮은 우선순위

### 7. More Visual Comparison Examples

- add examples for “current UI vs mockup” comparison prompts
- add examples for “repair only one region” prompts

### 8. Repository Polish

- consider platform README bilingual sections where helpful
- consider a short architecture note for how references, examples, and platform adapters relate
